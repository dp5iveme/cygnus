# Work OS: 사람이 사람다운 일을 하게 하는 플랫폼

**Version 5.1**
**2025년 12월**

*"사람이 사람다운 일을 하게 하는 Work OS"*

---

## 요약 (Executive Summary)

**[문제]**

사람들이 사람다운 일을 못하고 있습니다.

뭘 해야 하는지 몰라서 헤매고,
어떻게 해야 하는지 몰라서 시행착오를 반복하고,
인력이 부족해서 모든 걸 감당하고,
기업이 어떻게 돌아가는지 안 보여서 문제가 터져야 압니다.

작은 기업은 체계가 없어서 맨땅에 헤딩하고,
큰 기업은 규모가 커질수록 세밀하게 안 보이고, 느려지고, 곪아갑니다.

**[비전]**

우리는 사람이 사람다운 일을 할 수 있는 세상을 만들려고 합니다.

단순 반복은 시스템이,
부족한 역량은 AI와 생태계가,
기업은 실시간으로 보이게 하고,
사람은 오직 사람만이 할 수 있는 일에 집중합니다.

**[수단]**

이를 위해 **Work OS**를 만듭니다.

Work OS는 일하는 방식의 운영체제입니다:
- 업무를 규격화하고 (UBIP)
- 업무를 연결하고 (워크플로우)
- 부족한 것을 채우고 (AI Agent, 마켓플레이스)
- 기업을 실시간으로 보여주고 (온톨로지, 디지털 트윈)
- 가치를 측정합니다 (업무 경제)

**[결과]**

그 결과, 사람은 **"사람다운 일"**에 집중합니다:
- 창의적인 일
- 전략적인 일
- 관계를 만드는 일
- 의미를 찾는 일

## 1. 사람이 사람다운 일을 못하고 있다

### 1.1 현실: 사람다운 일에 집중하지 못하는 이유

이 프로젝트는 하나의 근본적인 질문에서 시작되었습니다:

> *"왜 사람들은 기계나 AI가 대신할 수 있는 일에 시간을 낭비하고 있는가?"*

현대 비즈니스 환경을 관찰하면서 발견한 문제들:

```
┌─────────────────────────────────────────────────────────────┐
│                사람다운 일에 집중하지 못하는 이유             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  문제 1: 뭘 해야 하는지 몰라서 헤맨다                        │
│  ─────────────────────────────────────────                  │
│  경영에 필요한 직무가 무엇인지 모른다                        │
│  어떤 역할이 필요한지, 어떤 역량이 필요한지 막막하다         │
│                                                             │
│  문제 2: 어떻게 해야 하는지 몰라서 시행착오를 반복한다       │
│  ─────────────────────────────────────────                  │
│  "고객 분석 좀 해줘" → "어떤 형식으로?" → "음... 그냥 엑셀?" │
│  (일주일 후) → "이게 아니라 이런 걸 원했는데..."             │
│  인터페이스가 명확하지 않아서 발생하는 불필요한 상황         │
│                                                             │
│  문제 3: 인력이 부족해서 모든 걸 감당한다                    │
│  ─────────────────────────────────────────                  │
│  HR, 마케팅, 재무, 개발... 모든 걸 1-2명이 감당              │
│  정작 중요한 일에 시간을 쓸 수 없다                          │
│                                                             │
│  문제 4: 기업이 어떻게 돌아가는지 안 보여서 문제가 터져야 안다│
│  ─────────────────────────────────────────                  │
│  월말 보고서로는 늦다                                        │
│  숫자만 있고 맥락이 없다                                     │
│  연결고리가 머릿속에만 있다                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

│                                                             │

│  ═══════════════════════════════════════════════════════    │

│  • 의사결정이 느리다                                         │

│                                                             │

```

### 1.3 대기업은 해결된 게 아니다

```
┌─────────────────────────────────────────────────────────────┐
│                 대기업이라고 해결된 것이 아니다               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  흔한 착각:                                                 │
│  "대기업은 자금이 있으니까 시스템도 있고 다 해결됐겠지?"    │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │

│    → 문제가 해결된 게 아니라 묻어둔 것                      │
│                                                             │
│  • 어디가 곪아가는지 모른다                                  │
│    → 월말/분기말 보고서로는 늦다                            │

│                                                             │

│                                                             │
└─────────────────────────────────────────────────────────────┘

## 5. 가치 측정: 모든 업무는 거래다

### 5.1 업무 경제 시스템의 필요성

```

├─────────────────────────────────────────────────────────────┤
│                                                             │
│  핵심 원칙:                                                 │
│  ═══════════                                                │
│  "기업의 모든 활동은 고객에게 가치를 제공하는 것이다.       │

│                                                             │

│                                                             │
│  원칙 1: 모든 업무는 상품이다                               │
│  • UBIP로 정의된 업무 = 쇼핑몰의 상품                       │
│  • 명확한 입출력(IO) = 상품 규격                            │
│  • 품질 기준 = 상품 스펙                                    │

│  • 업무 수락 = 결제                                         │

│  • 토큰 지급 = 정산                                         │
│                                                             │
│  원칙 3: 거래가 되면 가치가 측정된다                        │
│  • 가치 없는 업무 = 아무도 안 산다 → 자연 도태             │
│  • 가치 있는 업무 = 많이 팔린다 → 자연 성장                │
│                                                             │

```

## 14. 부록

### 14.1 용어 정의

| 용어 | 정의 |
|-----|-----|
| **Work OS** | 일하는 방식의 운영체제. 업무를 규격화하고, 연결하고, 자동화하고, 보여주는 플랫폼 |
| **사람다운 일** | 창의적, 전략적, 관계 중심의 일. AI나 시스템이 대신할 수 없는 일 |
| **UBIP** | Universal Business Interface Protocol - 업무의 입출력을 API처럼 표준화하는 프로토콜 |
| **업무 경제** | 모든 업무를 거래로 만들어 가치를 측정하고 낭비를 드러내는 시스템 |
| **실시간 가시성** | 기업 전체가 실시간으로 보이고, 예측되는 상태 |
| **AI Agent** | 특정 직무를 수행하는 AI 소프트웨어, 사용자의 "분신" |
| **온톨로지** | 특정 도메인의 개념과 관계를 형식적으로 정의한 지식 표현 체계 |

| **지식 그래프** | 개념(노드)과 관계(엣지)로 이루어진 지식 체계 |
| **워크플로우** | 업무들이 연결되어 자동으로 흐르는 프로세스 |
| **마켓플레이스** | 업무 서비스를 사고파는 플랫폼 |
| **매출 역산** | 외부 매출에서 시작하여 내부 업무별로 가치를 배분하는 방식 |
